T-01 AI 규정 적합성 선분석·판정초안 ★MVP

  • AS-IS <가설>: 심의역이 원고·기획안을 관련 규정(방송심의규정·표시광고법 등)과 전량 수작업 대조. 규정 개정 추적·유사 판례 검색이 개인 경험에 의존, 처리 시간 편차 크고 판정 일관성 리스크.
  • TO-BE: 접수된 원고를 AI가 규정에 자동 매핑·리스크 태깅(F-01) → 신뢰도 기반 3분기 라우팅(F-02: 자동초안/근거보강/에스컬레이션) → 판정·사유·근거 인용 포함 판정초안 자동 생성(F-03), 근거는 RAG 검색(F-04)·부족 시 재보강(F-05). 모든 판정에 근거·신뢰도·모델버전 감사 로그.
  • EXAMPLE [BM]: 홈앤쇼핑 PI — AI/BERT 기반 금지어·표현 검출로 심의 보조, "AI 보조 + 사람 확정" 운영.
  • 고려사항: 근거(조항·판례) 미첨부 초안 무효 원칙 / 고위험·신유형 자동확정 금지 / 규정 개정 시 지식베이스 동기화(5.4 연계) / 오탐·미탐 기준선과 신뢰도 임계값은 파일럿에서 캘리브레이션.

T-02 심의역 검토·확정 워크벤치 ★MVP

  • AS-IS <가설>: 판정 결과 정리·기안·근거 첨부가 문서 수작업. 초안-근거-원문을 오가는 화면 전환 비용이 커 검토 리드타임 증가.
  • TO-BE: 단일 검토 화면(63 화면설계·U 요소)에서 AI 판정초안(판정 배지·신뢰도 게이지·사유) + 근거 카드(조항·판례, 클릭 시 원문) + 원고 프리뷰(위반 구간 하이라이트)를 한눈에 보고 ［승인/수정 후 확정/에스컬레이션］(F-06·F-07). 사유 미입력 시 확정 불가 Rule 내장.
  • EXAMPLE [BM]: 홈앤쇼핑 PI — 심의 결과 화면에서 근거 구간 바로 확인, 확정은 사람.
  • 고려사항: 확정 없이 통보 불가(5.1.3 Rule) / 수정 유형·사유가 F-10 지식 보강 루프의 입력이 되도록 데이터 설계 / 접근권한(심의역·위원회) 분리.

T-03 접수·완결성 자동화 & 유형 라우팅

  • AS-IS <가설>: 접수 시 필수항목·메타 점검이 수작업 체크리스트. 심의 유형(원고/영상/상품) 구분이 암묵적이라 담당 배정·처리 경로가 사람 판단에 의존.
  • TO-BE: 접수 시 완결성 자동 점검·반려(F-08, A-5.1.1.1~2) + 유형 분류·라우팅 훅(A-5.1.1.3): MVP에선 원고 단일 유형의 경량 분류, Post-MVP에 F-11로 확장해 유형별 파이프라인 자동 라우팅.
  • EXAMPLE: 홈쇼핑 상품 QA의 대상분류→검사→판정 구조 차용 [BM].
  • 고려사항: 유형 분류 기준(무엇을 보고 어느 심의로 보내나)은 인터뷰 확정 항목 / 반려 사유 표준코드화(통계·개선 루프용).

T-04 결과 통보·판정 지식자산화

  • AS-IS <가설>: 확정 판정의 제작·편성 통보가 개별 연락 중심, 판정 이력이 문서로 산재해 판례로 재사용되지 못함.
  • TO-BE: 확정 판정·수정의견 자동 통보(F-09, A-5.1.3.2) + 판정 이력을 구조화 적재(F-10, A-5.1.3.3)해 5.4 판정기준·지식으로 환류 — RAG 근거(F-04)가 자기 판정 이력으로 계속 강해지는 보강 루프 완성.
  • EXAMPLE: 자사 판정 이력 기반 유사 판례 추천(T-01의 근거 품질이 누적적으로 향상).
  • 고려사항: 이력 스키마에 수정 전/후·사유 포함(모델 개선 신호) / 개인정보·민감정보 마스킹 / 통보 채널(그룹웨어 등)은 인터뷰 확정.

T-05 멀티모달 심의 확장 (영상·상품) — Post-MVP

  • AS-IS <가설>: 영상 심의는 쇼호스트 멘트를 사람이 들으며 확인(텍스트화 안 됨), 상품표시 검토는 이미지·상세페이지 육안 대조.
  • TO-BE: 유형 축 확장 — 영상=STT(멘트)+OCR(자막)(F-12·F-13) / 상품표시=OCR(F-13) 입력 어댑터를 추가하고 공통 판정엔진(T-01) 재사용. 검토 화면은 중앙 프리뷰만 유형별 교체(영상 플레이어+STT 싱크·이미지+OCR 오버레이). 영상심의 별도 Sub(가칭 5.1.4)는 이때 신설 검토.
  • EXAMPLE [BM]: 홈앤쇼핑 PI 3축 — STT(멘트)·AI(금지어)·OCR(상품 QA) / SK스토아 "멘트 클릭 시 영상 구간 이동".
  • 고려사항: MVP 구조를 흔들지 않는 확장(입력 어댑터 패턴 유지)이 전제 / STT·OCR 정확도 기준선 확보 후 착수 / 시점이 다른 영상심의(제작 後)의 프로세스 위치는 인터뷰 확정.
