610 프로세스정의서 — 5.1.2 규정 적합성 분석·판정

섹션 구성 = 프로세스 정의서 표준 템플릿 (SCM 5단 프레임워크·브레드크럼 + PI As-Is/To-Be + AI Logic 결합)
상태: <설계 가설> — 인터뷰 확정 예정. 벤치마크 인용은 [BM]로 표기(도입 여부·수준 미확정). 임계값은 D-02에서 확정.

브레드크럼 (계층 위치) ※SCM 메타표 채택 — 모든 정의서 상단 고정

Mega5. 심의Main5.1 사전심의
Sub5.1.2 규정 적합성 분석·판정유형프로세스 정의서 (L3)

상위: 5.1 사전심의 (610) · 밸류체인 위치: 사전기획→QA→편성/제작→심의(사전심의)→사후관리

1. 개요

  • 목적: 접수된 심의 대상(방송 원고·쇼호스트 멘트·상품 정보/영상)의 규정 적합성을 분석. AI 에이전트가 선분석·검토배정·근거 정리를 자율 수행하고 사람이 판정·확정.
  • 범위: 액티비티 5.1.2.1 ~ 5.1.2.5.
  • 핵심 원칙: AI 보조 + 사람 확정 — 고위험·신유형은 자동확정 금지. (R-01)

2. As-Is vs To-Be ※PI 과제정의서 틀 + 벤치마크

구분AS-IS (현행)TO-BE (개선)
심의 방식담당자 전적 수작업, 규정 기억·검색 의존AI 선분석 → 확신도 검토배정 → 근거 패킷, 사람 판정
멘트 처리쇼호스트 멘트 수기 텍스트화 ([BM] 3~5h·부정확)STT 자동 텍스트화 (데이터방송 전수 [BM])
규정 판정담당자 판단, 과거이력 수기 확인AI/NLP가 금지어·규정위반 후보 제시(라이브는 가이드 [BM])
상품 검토수기 검토OCR로 상품 이미지/자막 텍스트 추출 [BM]
근거/이력산발적조항·판례 자동 인용 + 감사로그

[BM] = 홈앤쇼핑 PI ⓕ 솔루션 심의 개선(STT/AI-BERT/OCR) 벤치마크. 도입 여부·수준은 미확정.

3. 액티비티 목록 (L4)

ID액티비티수행주체전환요약
5.1.2.1선분석·규정 매핑AI Agent★규정 매핑·리스크 태깅·완결성 반영
5.1.2.2확신도 검토배정AI Agent★확신도·리스크로 3분기(빠른 확인/근거 보강/에스컬레이션)
5.1.2.3판정 근거 구성AI Agent★위반 후보·근거 인용·확신도 정리 (판정 아님)
5.1.2.4근거 보강·재검색AI Agent근거 불충분 시 재검색 후 재라우팅
5.1.2.5상위 검토·에스컬레이션Human (심의위원회)신유형·고위험 건 사람 상위 검토

4. AI Logic 개요 (Input → Process → Output) ※SCM AI Logic 틀 + AI 심의 3축 [신규]

Input(입력)

  • 심의 대상: 방송 원고 · 쇼호스트 멘트(→STT 텍스트화 [BM]) · 상품 정보/이미지/자막(→OCR [BM])
  • 참조 지식자산: 규정·조항, 판례·피드백 이력, 과거 심의이력 (※수행주체 아님, 데이터 소스)

Process(처리)

  1. 선분석·규정 매핑(5.1.2.1) — NLP로 조항 매핑, 리스크 태깅, 완결성 반영
  2. 확신도 검토배정(5.1.2.2) — 확신도·리스크 → 3분기: 빠른 확인 / 근거 보강 / 에스컬레이션
  3. 판정 근거 구성(5.1.2.3) — 위반 후보·근거(조항/판례) 인용·확신도 (판정 아님)

모델·보조(AI 심의 3축, 벤치마크 기반)

축역할벤치마크 시사점 [BM]
STT멘트 자동 텍스트화정확도 한계→속기사 병용 가능, 데이터방송 전수 적용
AI/NLP금지어·규정위반 후보 판정과거 심의자료 학습 필요, 라이브는 "가이드 수준"(사람 확정)
OCR상품 이미지/자막 텍스트 추출해상도 따라 인식률 편차

입력 어댑터 = 유형별 <신규>: 5.1.1에서 라우팅된 유형에 따라 전처리가 갈리고, 공통 규정매핑·검토배정·근거·분석 엔진은 1개. → 근거·분석 엔진 재사용, 유형 확장은 어댑터만 추가.

심의 유형입력 어댑터MVP?
원고·기획NLP(텍스트 직접)★MVP
영상·완성물STT+OCR+영상분석 → 텍스트/태그Post-MVP
상품표시OCR → 텍스트Post-MVP

(상세: 680 로드맵이행계획)

Output(산출): 근거 패킷(위반 후보·근거조항/판례·확신도) → 사람 판정 · 감사로그(판정·근거·확신도·모델버전)

5. Rule (판정·라우팅 규칙)

  • 반복 재라우팅 N_max회 초과 시 강제 에스컬레이션 (R-03, N_max = D-02)
  • 고위험 카테고리(신유형·중대위반)는 사람 확정 필수 — 자동 확정 금지 (R-01)
  • 근거 조항·판례 미첨부 초안은 무효 처리 (R-04)
  • 모든 판정·근거·신뢰도·모델 버전은 감사 로그에 기록 (R-07)
