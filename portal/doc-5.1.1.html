상위: 5.1 사전심의 (610) · 다음: 5.1.2 규정 적합성 판정

1. 개요

  • 목적: 심의 대상을 접수하고 ① 완결성(필수항목·메타) 점검과 ② 유형 분류·라우팅을 수행해, 후속 판정(5.1.2)의 입력 품질과 처리 경로를 보장.
  • 범위: 접수 → 완결성 체크 → 유형 분류·라우팅.
  • 유형 축 도입(<신규>): 접수 시점에 심의 대상을 원고·기획 / 영상·완성물 / 상품표시·광고로 분류 → 유형별 판정 파이프라인으로 라우팅. (상세: 680 로드맵이행계획)

2. As-Is vs To-Be

구분AS-IS <가설>TO-BE
접수제작·편성에서 수기/분산 접수시스템 자동 접수·메타 수집
완결성담당자 육안 확인필수항목·메타 자동 점검, 누락 시 반려·보완
유형 구분암묵(채널·경험으로 구분)유형 자동 분류·라우팅(원고/영상/상품) <신규>

4. 분류 Logic (경량)

  • 유형 분류 = 접수 메타·포맷 기반 규칙 우선(예: 텍스트 원고 → 원고심의, 영상 파일 → 영상심의, 상품 이미지/표시 → 상품표시심의) + 모호 시 AI 분류 보조·사람 확인.
  • 유형별 필수항목 세트가 다름 → 완결성 체크는 유형 확정 후 세트 적용(또는 접수 채널로 선판별).

5. Rule

  • 필수 항목·메타 누락 시 접수 반려 후 보완 요청 (판정 단계 미진행) (R-08)
  • 접수 채널(제작 중 심의 / 방송 전 최종)·유형별 필수 항목 세트를 표준화
  • 유형 미상·경계 건은 사람 확인 후 라우팅 <신규>
  • 모든 접수·반려·분류 이력은 로그에 기록 (R-07)
