660 판정로직정의서 — Rule / AI Logic

판정 룰·AI 로직의 단일 정본. 상태 <설계 가설> — 파라미터는 파일럿·인터뷰에서 확정.
ID 규약: R-nn (발번 후 불변). 연결: R ↔ A(발동 액티비티) ↔ F(구현 기능) ↔ D(파라미터 자산). digital twin 예약: R-ID ↔ 룰 엔진 레코드(개발 트랙).

브레드크럼

Mega5. 심의Main5.1 사전심의
대상5.1 전체 (판정·라우팅·게이트·감사)유형판정로직 정의서

1. Rule 카탈로그

ID룰분류발동 지점구현파라미터
R-01AI 보조 + 사람 확정 — 최종 판정 책임은 사람. 고위험 카테고리(신유형·중대위반)는 자동확정 금지원칙/게이트A-5.1.2.2 · A-5.1.3.1F-02·F-06고위험 카테고리 목록(D-02)
R-02신뢰도 3분기 라우팅 — 신뢰도 ≥ T_auto → 자동초안 / 미만 → 근거보강 / 고위험·신유형 → 에스컬레이션라우팅A-5.1.2.2F-02T_auto(D-02, 가안 0.8)
R-03재라우팅 상한 — 보강 재라우팅 N_max회 초과 시 강제 에스컬레이션라우팅A-5.1.2.4F-05·F-07N_max(D-02)
R-04근거 필수 — 규정 조항·판례 미첨부 판정초안은 무효게이트A-5.1.2.3F-03·F-04—
R-05확정 사유 필수 — 사유 미입력 시 승인/수정확정 불가게이트A-5.1.3.1F-06—
R-06확정 전 통보 금지 — 사람 확정 없는 판정은 통보 대기열 진입 불가게이트A-5.1.3.2F-09—
R-07감사 전수 기록 — 모든 판정·근거·신뢰도·모델버전·라우팅 이력 감사 로그감사전 액티비티F-03·F-10보존 기간(인터뷰)
R-08접수 완결성 게이트 — 필수항목·메타 미충족 시 접수 반려(사유 코드 부여)게이트A-5.1.1.1~2F-08필수항목 정의(인터뷰)
R-09유형 라우팅 — 원고/영상/상품 판별 → 유형별 입력 어댑터 배정 (MVP=원고 단일 훅)라우팅A-5.1.1.3F-11유형 분류 기준(인터뷰) <가설·Post-MVP>
  • R-05·R-06은 검토 화면 프로토타입(S-01)에서 동작 실증 완료.
  • 파라미터 3종(T_auto·N_max·고위험 목록)의 정본 = 데이터정의서 D-02, 확정 = 파일럿 캘리브레이션.

2. AI Logic — 공통 판정엔진 (Input → Process → Output)

Input: 심의 대상 텍스트(유형 어댑터 산출) + 참조 지식자산(규정·조항, 판례·판정 이력 — 5.4 공급)

Process:

  1. 선분석·규정 매핑 (A-5.1.2.1, F-01) — NLP 조항 매핑·리스크 태깅
  2. 신뢰도 라우팅 (A-5.1.2.2, F-02) — R-02 적용, 근거 검색(F-04) 결합
  3. 판정초안 생성 (A-5.1.2.3, F-03) — 판정·사유·근거 인용·신뢰도, R-04 게이트
  4. (조건) 근거 보강·재검색 (A-5.1.2.4, F-05) — R-03 상한

Output: 판정초안(판정/사유/근거/신뢰도) + 감사로그(R-07)

3. 유형별 입력 어댑터 (판정엔진 앞단 — 엔진은 1개)

유형어댑터스코프

상품표시·광고OCR → 텍스트Post-MVP (F-13) [BM]
