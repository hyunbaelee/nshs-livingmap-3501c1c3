610 프로세스정의서 — 5.1 사전심의 (Main overview)

5.1 사전심의 수직 슬라이스 통합 뷰. 상태 <설계 가설> — 인터뷰 확정 예정. MVP 스코프 = 원고 텍스트 심의(PRD 60, 개발 트랙).

브레드크럼

Business Area홈쇼핑 밸류체인Mega5. 심의
Main5.1 사전심의유형Main 정의서 (L2)

밸류체인 위치: 사전기획 → QA → 편성/제작 → 심의(사전심의) → 사후관리

1. 개요

방송 전 심의. 심의 대상을 접수·분류하고, 규정 적합성을 AI가 선분석·근거 정리, 사람이 판정·최종 확정해 제작·편성에 통보. MVP는 원고·기획(텍스트) 심의에 집중.

2. 전체 흐름 (Sub 3묶음 + 유형 축)

[5.1.1 접수·완결성·유형분류]
      │  (원고 / 영상 / 상품표시 라우팅)
      ▼
[5.1.2 규정 적합성 분석·판정]  ← 유형별 입력 어댑터(NLP/STT·OCR/OCR) + 공통 근거·분석 엔진
      │  선분석 → 확신도 검토배정 → 빠른 확인 / 근거 보강 / 에스컬레이션
      ▼
[5.1.3 검토·확정·통보]  ← 사람 최종 확정 → 통보 → 이력·지식 적재
      │
      └──▶ 5.4 판정기준·지식 (보강 루프)

4. 유형 축 & MVP 경계

심의 유형AI 역량스코프
원고·기획(텍스트)NLP 규정매핑★MVP
영상·완성물STT+OCR+영상분석Post-MVP
상품표시·광고OCRPost-MVP

→ 현 MVP가 텍스트 중심인 것은 의도된 경계(누락 아님). 유형 축은 구조를 가로지르는 파라미터라, Post-MVP 확장 시 입력 어댑터만 추가(근거·분석 엔진·구조 재사용). (상세: 680 로드맵이행계획)

5. 공통 원칙

  • AI 보조 + 사람 확정 — 고위험·신유형 자동확정 금지, 최종 책임=사람. (R-01)
  • 모든 판정·근거·신뢰도·모델버전 감사 로그. (R-07)
  • 근거(조항·판례) 미첨부 초안 무효. (R-04)

6. 연계

  • 5.4 판정기준·지식(온톨로지·RAG) → 5.1.2 AI 판정 근거 공급 + 5.1.3 이력 → 5.4 보강.
  • 5.2 사후심의 / 5.3 대외대응 = MVP Out-of-scope.
  • PRD: 60 심의 AI PRD (개발 트랙) — U-1외부~U-5외부 ↔ 5.1.2 액티비티.
